You want both vectors to stay aligned row-by-row after unnesting.

---

## 🔹 Correct approach: unnest both columns together

Use `unnest_longer()` on both columns **in the same pipeline**, or better, unnest multiple columns at once.

### ✅ Best tidyverse way

```r id="n2k8qp"
library(dplyr)
library(tidyr)

df_long <- df %>%
  unnest_longer(c(vector1, vector2))
```

This guarantees:

* `vector1[i]` stays paired with `vector2[i]`
* rows expand in lockstep

---

## 🔹 If you used separate unnest calls (wrong way)

❌ This breaks alignment:

```r id="x8p2ld"

```

Because the second unnest is applied after the first expansion, causing mismatch.

```r id="k3m9vd"
df_long <- df %>%
```

This is useful if:

* vectors are nested unevenly
* you want guaranteed pairing logic

## 🔹 Key rule (important)

When working with list-columns in R:

Me: I am trying to mutate a column based on another column with fractional values as chr, example "1/2" I am using replace_na(eval(parse(text = portion)), 1) within the mutate but it is giving me .3333 for every value. Fix
The issue is your use of `eval(parse(...))` inside `mutate()`. That’s not just unnecessary — it’s also changing how the expression is being recycled and evaluated, which is why you’re getting a constant value like `0.3333`.

What you actually want is:

1. convert `"1/2"` (character) → numeric fraction
2. replace `NA` with `1`
3. do it row-wise safely

---

## 🔴 Correct approach (no `eval`, no `parse`)

Use a proper parser for fractions:

```r id="p8k2ld"
library(dplyr)

df <- df %>%
  mutate(
    portion_num = ifelse(
      is.na(portion),
      1,
      sapply(strsplit(portion, "/"), function(x) as.numeric(x[1]) / as.numeric(x[2]))
    )
  )
```

---

## 🔹 Why your version failed

```r
replace_na(eval(parse(text = portion)), 1)
```

Problems:

* `parse()` + `eval()` is not vectorized safely inside `mutate()`
* it evaluates in the wrong environment
* recycling leads to a single computed value being reused → your `0.3333`

---

## 🔹 Cleaner tidyverse version (recommended)

```r id="v2m9qp"
library(dplyr)
library(purrr)
library(tidyr)

df <- df %>%
  mutate(
    portion_num = map_dbl(portion, function(x) {
      if (is.na(x)) return(1)
      parts <- strsplit(x, "/")[[1]]
      as.numeric(parts[1]) / as.numeric(parts[2])
    })
  )
```

---

## 🔹 Even safer (handles bad strings)

```r id="k9p0sd"
df <- df %>%
  mutate(
    portion_num = map_dbl(portion, function(x) {
      if (is.na(x) || !grepl("/", x)) return(1)
      parts <- strsplit(x, "/")[[1]]
      as.numeric(parts[1]) / as.numeric(parts[2])
    })
  )
```

---

## 🔹 Key takeaway

* ❌ Avoid `eval(parse())` in `dplyr`
* ❌ It breaks row-wise evaluation
* ✅ Use `map_dbl()` or `sapply()` for safe per-row parsing
* ✅ Explicitly split `"a/b"` strings instead

## ✅ Tidyverse solution (robust)

```r id="t9k2qp"
library(dplyr)
library(purrr)
library(stringr)

df <- df %>%
  mutate(
    portion_num = map_dbl(portion, function(x) {
      if (is.na(x)) return(1)

      # if it's already a whole number like "1"
      if (!str_detect(x, "/")) return(as.numeric(x))

      parts <- str_split(x, "/", simplify = TRUE)
      as.numeric(parts[1]) / as.numeric(parts[2])
    })
  )
```
